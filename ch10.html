d212.東東下階梯
我們來推導這個問題，第一階只由一種走法，第二階可以走兩個一步或一個兩步，是兩種走法。那第三階呢?我們先想想怎麼走到第三階，一種是從第一階走兩步，或著從第二階走一步，總共是(1+2)=3總走法。
公式:
f(0)=1,f(1)=1
f(n)=f(n-1)+f(n-2) (n>=2)
其實

以下有兩種寫法
(1)直接用一個迴圈跑一次
(2)進入副程式計算
#include <iostream>
using namespace std;
#define N 10000000

int main(){
    int a[N];
    a[0] = a[1] = 1;
    for (int i = 2; i < N; i++)
        a[i] = a[i - 1] + a[i - 2];
      int n; //第n階
    while (cin >> n)
        cout << a[n] << endl;
}		 #include <iostream>
using namespace std;
#define N 10000000

int query(int i){
    if (i < 2)return 1;
    return query(i - 1) + query(i - 2);
}

int main(){
    int n; //第n階
    while (cin >> n)
        cout << query(n) << endl;
}
這兩種寫法答案都是可行的，接下來要考慮效率
首先第一種是先把答案儲存在陣列內，當他問你時立即輸出答案，時間複雜度是(1)
第二種是當他問你時，在去算答案，我們輸入測資試試看，發現到40開始就跑很慢，為什麼?
當你要找 query(40)時要先找 query(39)和 query(38)，要找 query(39)時要先找 query(38)和 query(37)，以此類推.....
note :如果答案(會用到的值)可以先求，先儲存，提高程式效率
